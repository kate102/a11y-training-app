Mission Statement

Accessibility, in this context, is about the user experience of using your site with the assistive technology tools that they need.

Fact 1. Assistive technology users will simply leave the site if the hurdles become too arduos.

Fact 2. Assistive technology only fails when the underlying code isn't written properly.

Fact 3. Implementing accessibility enablement is straight forward if you do it from the beginning.

Fact 4. Many developers don't realise the implications of the coding decisions that they take when it comes to assistive technology.
